Circular Iteration

Purpose

This system provides an end-of-session documentation maintenance workflow. Unlike the Initialization Prompt (which prepares the AI for a new session), this process focuses solely on updating our documentation to reflect the latest changes and achievements. Run this after completing significant work to ensure our documentation stays current and comprehensive.

Documentation Refresh Prompt

I need you to help me update our documentation based on today's session. Here's what I need: 1. First, review the Developer Overview (developer_overview.html): - Update the project description if our goals or scope have evolved - Add any new technical components or dependencies - Refresh the implementation details with our latest approaches - Update the current status and next steps 2. Then, create a new Recent Methods entry (recent_methods_X.html): - Summarize today's key achievements and changes - Document any challenges we encountered and how we solved them - List any important decisions or architectural changes - Include any new patterns or approaches we developed - Add code snippets for significant changes - Link to relevant files and PRs 3. Finally, ensure all documentation maintains: - Our consistent styling and formatting - Clear navigation between pages - Up-to-date cross-references - Proper version history Please proceed with these updates while maintaining our established documentation standards and visual design.

Usage Instructions

  1. Run this prompt at the end of each significant development session
  2. Provide context about the day's achievements and changes
  3. Review and approve the suggested documentation updates
  4. Ensure all cross-references are maintained
